Social Network

GitHub link

Social Network is a large single-page app built with a modern stack of technologies including TypeScript, JavaScript, React, Redux, HTML, CSS3, SASS, Material-UI, and Jest for testing.

During my participation in the IT-Incubator front-end course, I had the opportunity to create this app and apply my knowledge in a practical setting, using contemporary coding practices and techniques.

After completing the course, I recognized the importance of code predictability and reducing potential mistakes in future development. To address this, I made the decision to refactor the existing JavaScript codebase into TypeScript. This transition allowed for stronger typing and improved overall code quality and reliability.

Furthermore, I ensured the correctness of the code by writing tests using the Jest library. These tests provided confidence that the expected results of code execution were accurate, enhancing the robustness of the application.

  • The functionality of the app includes: 
  • - Cookie-based authentication with controlled forms for secure user authentication.
  • - Personal page where users can view and modify their information, including the ability to change their profile photo.
  • - Users page with features such as subscriptions, pagination, filtering by name, and sorting based on subscription status.
  • - Chat page implemented using the JavaScript library Socket.IO, enabling real-time communication between participants. The chat displays participant names and profile photos alongside the messages.
  • - Weather page that allows users to check the current weather based on the input city.
  • - And more additional features.

Finally, the project was deployed on AWS, utilizing the cloud platform's infrastructure to host the application.

Project design

The design was created in Figma to establish a clear visual structure for every page and to obtain precise parameters for the elements.

Development

To ensure easy scalability of the app, I chose Redux for state management. Initially, I implemented class components and lifecycle methods to handle component behavior. However, I later refactored the code to functional components and utilized React Hooks to improve readability, testability, and debugging capabilities. To handle routing and keep the UI in sync with the browser URL, I employed React Router. Additionally, I implemented a feature that displays error messages for HTTP response codes 400 and 500, providing users with relevant feedback. To enhance user experience, a pre-loader GIF is displayed before the app is rendered, indicating that the app is loading and improving overall usability. Throughout the development process, I encountered numerous challenging tasks, which deepened my understanding of application workflows and design patterns. This experience helped me reduce mistakes and make the codebase more predictable and robust.
